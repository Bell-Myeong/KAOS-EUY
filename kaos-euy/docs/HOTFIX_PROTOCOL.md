# 핫픽스/릴리즈 프로토콜

## 핫픽스 정의
- Sev1: 주문/업로드/어드민 핵심 기능이 중단
- Sev2: 일부 기능 저하 또는 지속적 오류 증가

## 승인 규칙
- 최소 2인 승인 (개발 1인 + 운영/PM 1인)
- 긴급 상황에서도 승인 기록은 남긴다

## 브랜치/태그/릴리즈 노트
- 브랜치: `hotfix/YYMMDD-short-desc`
- 태그: `hotfix-YYMMDD` 또는 `release-YYMMDD`
- 릴리즈 노트는 변경 요약 3줄 이내로 유지

## 배포 절차
- 스테이징 재현 → 수정 → `scripts/verify-deploy.mjs` 실행 → 프로덕션 배포
- 배포 순서와 체크는 `docs/DEPLOY_RUNBOOK.md` 기준으로 진행

## 롤백 기준
- 5xx 급증
- 주문/업로드 불가
- admin 접근 불가
- 롤백이 더 빠르면 롤백 우선

## 안전 규칙
- 데이터 마이그레이션이 필요한 변경은 런칭 직후 48시간 내 금지
- 사용자 데이터에 영향을 주는 변경은 반드시 사전 공지
- 환경 변수 변경은 배포 전 이중 확인
